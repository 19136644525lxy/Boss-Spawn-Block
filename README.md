# Boss Spawn Block-Bsb（自定义生物刷怪笼-原名自定义Boss刷怪笼）

## 简介
这个模组添加了可自定义的 Boss 刷怪笼。也不能说单只能BOSS，是个生物实体就行

模组灵感来源于 暮色森林 的刷怪箱，功能基本一致，只不过本模组可以通过配置文件修改刷出的生物；

刷怪笼在除了生存模式以外的模式都不会激活，当玩家处于生存模式并在刷怪笼9*9*9的范围内将激活生成机制。

注: 刷怪笼刷新生物后就消失，做专属boss刷怪笼的时候注意表明。

## 配置
0.2.0-rc-1版本过后，原本在创造物品栏内的刷怪笼改为示例刷怪笼（不可进行配置）；

新的刷怪笼采用标签的形式来进行配置，

所有的新刷怪笼配置文件都可以在.minecraft/config/bsb文件夹内找到，配置文件命名格式为：自定义标签.toml。

注:common.toml配置文件修改不会有任何效果，但也不要删除

## 指令
BSB的自定义命令：

/bsb help - 显示帮助信息。

/bsb list - 列出所有创建的Boss刷怪笼。

/bsb create <标签> [实体刷怪蛋ID/中文名称] [描述] - 创建一个带有标签的新Boss刷怪笼（中文描述详情看映射表板块）。

/bsb give <标签> - 将带有对应标签的Boss刷怪笼给予玩家。

/bsb delete <标签> - 删除对应标签的刷怪笼。

注：使用 /reload 指令可能导致 /bsb 指令失效，需重启游戏（整合包测试中发现）。

PS：若新创建的刷怪笼所刷新的生物ID不合法或不规范则不会在生存模式启动，聊天框会弹报错信息提醒检查配置文件

## 映射表
映射表在0.2.0-rc-2版本加入，文件存放位置为.minecraft/config/bsb文件夹内，名称为mapping.json；

映射表的作用为：在使用/bsb create指令时，在标签后的描述就可以使用映射表的预设值来进行填写；

例如：/bsb create 1 猪，这个指令会在映射表找到“猪”的值进行创建，就不用填写ID啥的了（当然你一步步来也行）；

映射表格式：

{

"中文描述":"生物刷怪蛋ID"

}

后续将添加英文映射表。
